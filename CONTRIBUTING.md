# 貢獻指南

感謝您對本專案的關注！

## 專案性質

本專案為個人文件儲存庫，主要用於個人文件管理。

## 貢獻方式

由於這是個人私有專案，目前不開放外部貢獻。

## 文件組織

如需新增文件，請遵循以下原則：

1. **文件命名**: 使用清晰、描述性的檔案名稱
2. **目錄結構**: 將相關文件放在適當的目錄中
3. **文件格式**: 優先使用 Markdown (.md) 格式以便版本控制
4. **提交訊息**: 使用清晰的中文描述變更內容

## 文件更新流程

1. 確認要新增或修改的文件
2. 將文件放置在適當目錄
3. 提交變更並撰寫清楚的提交訊息
4. 推送到遠端儲存庫

## 注意事項

- 不要上傳包含敏感個人資訊的文件
- 確保文件格式正確且可讀
- 定期整理和歸檔舊文件
- 使用 .gitignore 排除不需要的檔案

## 聯絡資訊

如有任何問題，請透過 GitHub Issues 提出。
